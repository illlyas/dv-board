# V2 看板生成系统使用指南

## 🎉 V2 已集成到应用中！

V2 版本现在是**默认版本**，当你访问应用时会自动使用 V2 的新流程。

## 🔄 版本切换

### 在界面中切换
- 应用右上角有**版本切换按钮**
- 点击可以在 V1 和 V2 之间切换
- 切换后会清空当前状态，重新开始

### 修改默认版本
如果你想修改默认使用的版本，编辑 `src/app/page.tsx`：

```typescript
// 修改这一行来改变默认版本
const [version, setVersion] = useState<"v1" | "v2">("v2"); // "v1" 或 "v2"
```

## 🚀 使用 V2 生成看板

### 第 1 步：输入需求
在输入框中描述你想要的数据看板，例如：
```
创建一个电商销售数据看板，包含：
- 销售额趋势图
- 品类占比分析
- TOP10 商品排行
- 关键指标卡片（GMV、订单量、客单价）
```

### 第 2 步：AI 并发设计
点击提交后，AI 会**同时**执行两个步骤：

#### ① VI 系统设计
- 自动设计完整的视觉标识系统
- 包含颜色、字体、间距、圆角等 Design Token
- 参考 shadcn/ui 的设计规范
- 支持暗色/亮色主题

#### ② 看板故事设计
- 分析业务需求，规划页面结构
- 定义每个页面的分析目标和关键问题
- 建议必要的内容模块（图表、KPI、表格等）
- 设计数据叙事逻辑

### 第 3 步：生成 JSX 代码
前两步完成后，AI 会根据 VI 系统和看板故事：
- 生成完整的 React 组件代码
- 使用 `React.createElement()` 格式（无需转译）
- 应用 VI Token 到所有样式
- 内置模拟数据
- 支持多页面切换

### 第 4 步：实时预览
- 左侧显示设计过程（VI 系统、看板故事、代码）
- 右侧实时渲染最终效果
- 可以复制代码或下载为文件

## 📊 V2 vs V1 对比

| 特性 | V1 | V2 |
|------|----|----|
| **生成步骤** | 3步串行 | 2步并发 + 1步串行 |
| **VI 系统** | 简化版 | 完整 Design Token 体系 |
| **最终输出** | 结构化数据 | 可执行的 JSX 代码 |
| **预览方式** | 自定义渲染器 | 动态 JSX 渲染 |
| **设计参考** | 自定义 | shadcn/ui 风格 |
| **生成速度** | 较慢（串行） | 较快（并发） |

## 🎨 V2 的核心优势

### 1. 完整的 VI Token 系统
```typescript
{
  colors: {
    background: "#0a0e1a",
    foreground: "#e2e8f0",
    primary: "#3b82f6",
    accent: "#06b6d4",
    chartPalette: ["#3b82f6", "#06b6d4", ...],
    // ... 更多颜色
  },
  typography: {
    fontFamily: { display, body, mono, numbers },
    fontSize: { hero, h1, h2, h3, body, small, caption },
    // ... 更多排版
  },
  // ... 间距、圆角、阴影等
}
```

### 2. 直接生成可执行代码
不再需要中间转换步骤，AI 直接输出：
```javascript
export default function Dashboard() {
  const [page, setPage] = React.useState(0);
  
  return React.createElement("div", {
    style: { width: 1920, height: 1080, ... }
  }, ...);
}
```

### 3. 并发执行提速
Step 1 (VI 设计) 和 Step 2 (故事设计) 同时执行，节省约 30-50% 的时间。

### 4. 更专业的设计输出
- 参考 shadcn/ui 的设计规范
- 完整的组件风格指南
- 统一的视觉语言
- 更好的可维护性

## 🔧 API 端点

### V2 API
```bash
# Step 1: 设计 VI 系统
POST /api/board/v2/design-vi
Body: { "brief": "用户需求描述" }

# Step 2: 设计看板故事
POST /api/board/v2/design-story
Body: { "brief": "用户需求描述" }

# Step 3: 生成 JSX 代码
POST /api/board/v2/generate-jsx
Body: {
  "brief": "用户需求描述",
  "viSystem": { ... },
  "boardStory": { ... }
}
```

### V1 API（保持不变）
```bash
POST /api/board/analyze      # 需求分析
POST /api/board/structure    # 页面结构
POST /api/board/visualize    # 视觉设计
```

## 💡 使用技巧

### 1. 提供详细的需求描述
越详细的描述，AI 生成的结果越准确：
```
✅ 好的描述：
"创建一个电商运营监控大屏，需要展示：
1. 实时销售额和订单量（大数字 KPI）
2. 近30天销售趋势（折线图）
3. 品类销售占比（饼图）
4. TOP10 热销商品（排行榜）
5. 区域销售分布（地图或柱状图）
目标受众是运营团队，需要一目了然看到关键指标。"

❌ 不好的描述：
"做一个销售看板"
```

### 2. 指定视觉风格
可以在需求中指定想要的视觉风格：
```
"使用深色科技风格，蓝色系为主色调"
"采用清爽的亮色主题，适合汇报场景"
"高管汇报风格，正式庄重，使用金色高亮"
```

### 3. 明确数据维度
告诉 AI 你关心的数据维度：
```
"需要按时间、区域、渠道三个维度分析"
"重点关注同比、环比变化"
"需要展示目标达成率"
```

### 4. 复制代码使用
生成的 JSX 代码可以：
- 直接复制到你的 React 项目中
- 修改模拟数据为真实 API 调用
- 调整样式和布局
- 添加交互功能

## 🐛 常见问题

### Q: V2 生成失败怎么办？
A: 
1. 检查网络连接和 AI 服务状态
2. 尝试简化需求描述
3. 点击"重试"按钮重新生成
4. 如果持续失败，切换回 V1 版本

### Q: 生成的代码无法渲染？
A: 
1. 检查浏览器控制台的错误信息
2. 确保代码中没有语法错误
3. 检查是否使用了不支持的 API
4. 尝试重新生成

### Q: 如何修改生成的代码？
A:
1. 点击"复制代码"按钮
2. 粘贴到你的代码编辑器
3. 修改样式、数据、布局等
4. 在你的项目中使用

### Q: V1 和 V2 可以互相转换吗？
A: 不可以。V1 和 V2 使用完全不同的数据结构和生成流程，无法直接转换。

### Q: 哪个版本更好？
A:
- **V2 推荐用于**：新项目、需要完整 VI 系统、追求生成速度
- **V1 推荐用于**：需要精确控制布局、已有 V1 项目、需要兼容旧数据

## 📝 示例需求

### 示例 1: 销售数据看板
```
创建一个销售数据分析看板，包含3个页面：

第1页 - 销售总览：
- 本月销售额、订单量、客单价（KPI卡片）
- 近30天销售趋势（折线图）
- 品类销售占比（饼图）
- TOP10商品排行（表格）

第2页 - 区域分析：
- 各区域销售额对比（柱状图）
- 区域增长率排名（排行榜）
- 重点区域明细（表格）

第3页 - 渠道分析：
- 渠道销售额占比（环形图）
- 渠道转化漏斗（漏斗图）
- 渠道效率对比（表格）

使用深色科技风格，蓝色系主色调。
```

### 示例 2: 用户增长看板
```
创建一个用户增长分析看板，目标受众是产品团队：

核心指标：
- DAU/WAU/MAU 趋势
- 新增用户数和留存率
- 用户活跃度分布

分析维度：
- 按时间：日、周、月
- 按渠道：自然增长、广告、推荐
- 按用户类型：新用户、老用户、流失用户

视觉风格：清爽的亮色主题，适合产品评审会议。
```

### 示例 3: 运营监控大屏
```
创建一个实时运营监控大屏，1920×1080全屏显示：

实时指标（大数字）：
- 在线用户数
- 今日订单量
- 今日GMV
- 服务器负载

趋势图表：
- 24小时订单量趋势
- 实时流量曲线

异常监控：
- 告警列表（最近10条）
- 异常订单明细
- 系统状态指示

使用深色背景，高对比度，适合大屏展示。
```
