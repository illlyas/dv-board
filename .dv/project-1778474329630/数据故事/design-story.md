# 智慧工厂运营看板

## 概述
一个聚焦生产、设备、能耗、质量与运营的综合性可视化大屏，为设备与能耗管理人员提供实时监控、异常诊断与决策支持，助力工厂实现高效、稳定、低碳生产。看板采用深色科技风，适配2560×1080像素长屏，信息层级清晰，数据更新秒级。

## 业务目标
- **核心目标**：实时掌控工厂产、设、能、质、本等关键KPI，快速定位异常并驱动主动干预。
- **受众**：设备/能耗管理人员（以及生产管理人员、质量管理人员等协同角色）
- **决策支持**：支持排产调整、设备维护计划、节能措施启动、质量追溯与叫停、设备报警派工、成本异常审查等决策。

## 分析维度
- **时间维度**（小时/班次/日/周/月）：趋势分析，动态切换
- **产线/车间维度**：按产线对比，下钻至工位
- **设备/工位维度**：设备级OEE与故障分析
- **产品品类/型号维度**：质量、产量按产品细分
- **工序/工艺维度**：不良率按工序定位
- **班组/人员维度**：产量与停机关联班组
- **地域/厂区维度**：多厂区对比（如适用）

## 对比分析
- **同比**：与去年同期对比（年周期）
- **环比**：与上期（上一小时/班次/日）对比
- **目标对比**：与计划值/定额对比
- **基准对比**：与行业均值或历史最优对比
- **无需对比**（部分纯数值展示，如缺陷分类计数）

## 关键决策场景
1. **排产调整**：如果产量或计划完成率连续2小时低于目标90%，则触发排产优化建议，提示调整班次或产线任务
2. **设备维护**：如果OEE低于目标值（如85%），则建议安排预防性维护；若故障率上升10%则生成维修工单
3. **节能启动**：如果能耗总量或单位能耗超限（超计划10%），则自动启动节能措施（如调速、关闭非必须设备）
4. **质量叫停**：如果不良率超过设定阈值（如3%），则追溯缺陷来源，并发出停线建议，直至原因澄清
5. **设备报警派工**：如果设备报警事件发生（如温度/振动超限），则立即生成工单派发给对应维修班组
6. **成本异常审查**：如果生产成本环比上升超过5%，则审查物料消耗与损耗清单，并对比物料单价波动

## 预警规则
- 产量/计划完成率：低于目标值90%持续30分钟 → 黄色预警；低于80% → 红色预警
- OEE：低于85% → 黄色；低于75% → 红色
- 能耗超限：超计划10% → 黄色；超20% → 红色
- 不良率：高于1% → 黄色；高于3% → 红色（同时推送停线建议）
- 设备故障率：高于5% → 黄色；高于10% → 红色
- 成本异常：环比上升5% → 黄色；上升10% → 红色

## 页面规划建议
建议分为3个页面，按照总览→详情→诊断的逻辑展开，每页图表/表类组件（除KPI外）不少于5个：

1. **生产与效率总览**：左侧（左核心区）展示产量、计划完成率、OEE的实时卡片组（metric-group-inline）及趋势图；中栏展示各产线产能对比热力图；右栏展示设备故障率排行表与停机时长TOP10。此处承载5+个图表组件（热力图、排行表、趋势图等）。
2. **能耗与质量监控**：左侧展示能耗总量与单位能耗的时序图（叠加计划线与阈值预警），以及能耗构成饼图；中栏展示产品合格率/不良率趋势及缺陷分类堆叠图；右栏展示质量按工序/班组的对比柱状图以及原材料消耗趋势。此处承载6+个图表组件。
3. **设备与运营诊断**：左侧展示设备状态矩阵（OEE、故障率、运行状态）及报警事件滚动列表；中栏展示成本/利润趋势与分析图；右栏展示库存周转率、班组绩效表及原材料消耗排行。此处承载5+个图表组件。

**补充说明**：若某个页面实际图表组件数不足5个，可在生成时通过脚注说明取舍原因（例如“D页建议将三个表合并为两个表以聚焦”），当前规划均满足约束。

## 数据故事线
从宏观生产进度入手（产量与计划完成率），再下探到设备效能（OEE）与能耗水平，当发现偏差时进入质量与故障细节分析（不良率、缺陷分类、停机原因），最后评估成本与运营健康度（成本、利润、库存），形成“看全局→查原因→定措施”的闭环叙事逻辑。每页均设置预警触发点，引导用户关注异常数据，推动即时决策。
